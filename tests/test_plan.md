# Checklist Breve de Validación

## 1) Checklist de pruebas mínimas
- [ ] Probé al menos un caso exitoso del flujo principal.
- [ ] Probé al menos un caso con entrada inválida.
- [ ] Verifiqué que los mensajes de salida sean claros.

## 2) Checklist de documentación
- [ ] Completé `docs/propuesta.md`.
- [ ] Completé `docs/caso_de_uso.md`.
- [ ] Completé `docs/plan_de_pruebas.md`.
- [ ] Revisé coherencia entre propuesta, caso de uso y pruebas.

## 3) Checklist de ejecución
- [ ] Ajusté `scripts/run.sh` a mi lenguaje.
- [ ] Ejecuté el proyecto localmente sin dependencias complejas.
- [ ] Confirmé que el alcance sigue siendo pequeño.

## 4) Checklist de entrega final
- [ ] Mi repositorio mantiene la estructura solicitada.
- [ ] No usé frameworks, nube, APIs externas ni bases de datos.
- [ ] Incluí solo archivos necesarios y limpios.
- [ ] Mi propuesta es viable para una práctica corta.
